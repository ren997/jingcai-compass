# 竞彩罗盘开发任务草案

## 当前目标

围绕 MVP 闭环，优先打通“比赛池同步 -> 预测发布 -> 锁定 -> 自动结算 -> 历史展示”。

## 第零阶段：数据源可行性验证

- 注册 The Odds API 免费档并记录真实额度计算方式
- 在中国大陆部署节点验证中国体彩网和候选网关接口
- 连续两周保存体彩比赛池、赛果和亚盘样本
- 建立联赛、球队和比赛的双源映射样例
- 统计亚盘覆盖率、自动映射准确率、延迟和月度额度消耗
- 补测 API-Football 或商业样例源覆盖缺口
- 确认数据缓存、模型训练和页面展示的授权边界
- 按 `data-sources.md` 的 Go / No-Go 标准形成选型结论

完成条件：体彩源和亚盘源均通过验收，否则不进入完整业务开发。

## 第一阶段：基础骨架

- 明确 MVP 范围并冻结需求文档
- 设计核心表结构与枚举
- 设计 Provider Adapter、原始响应、比赛映射和盘口快照表
- 搭建前后端基础工程
- 建立提交规范、开发命令和文档入口

## 第二阶段：核心业务闭环

- 实现已选体彩 Provider 的比赛池和赛果同步
- 实现已选 Asian Odds Provider 的赛前盘口快照同步
- 实现双源比赛映射和低置信度人工复核队列
- 实现预测生成、发布与锁定流程
- 建立快照与审计日志基础能力
- 打通结算任务的输入输出结构

## 第三阶段：前台展示

- 实现首页核心指标卡片
- 实现比赛列表页与比赛详情页
- 实现历史记录页的基础筛选能力
- 实现统计分析页的 MVP 指标展示

## 第四阶段：联调与验收

- 对齐前后端字段命名和接口边界
- 完成锁定、防篡改、自动结算的验收用例
- 补齐风险提示、免责声明和隐私说明入口
- 准备首轮可演示版本
